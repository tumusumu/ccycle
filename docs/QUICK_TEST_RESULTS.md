## 🎯 关键测试场景

### 场景1：新用户完整流程
```
1. 访问 https://suton.cn
2. 点击"立即注册"
3. 填写：testuser0204 / test123456
4. 点击"注册"
5. → 跳转到 onboarding
6. 填写：1990 / 男 / 75kg / 20%
7. 点击"下一步"
8. → 跳转到 /plan/new
9. 选择今天
10. 点击"创建计划"
11. → 跳转到 dashboard
12. 点击"早餐"卡片
13. 填写：40g燕麦 / 2个全蛋 / 3个蛋白
14. 点击"✓ 确认完成"
15. → 返回 dashboard，进度环更新
16. 验证：碳水/蛋白质/脂肪/热量都有数据
```

### 场景2：历史记录补充
```
1. 访问 /plan (日历)
2. 点击昨天的日期
3. → 弹出 Modal，显示"这一天还没有记录"
4. 点击"📝 补充录入打卡"
5. → 跳转到 /history/2026-02-03
6. 验证：显示6个任务 + 控糖打卡卡片
7. 点击"早餐"卡片
8. → 跳转到 /meals/breakfast?date=2026-02-03
9. 填写数据并保存
10. → 返回 /history/2026-02-03
11. 验证：早餐状态变为"✓ 已完成"
12. 点击控糖打卡（🍎 没有吃水果）
13. → 勾选框变绿
14. 点击"返回日历"
15. → 返回 /plan
16. 验证：该日期显示为已记录状态
```

### 场景3：控糖打卡
```
1. 访问 /history/2026-02-03
2. 滚动到"控糖打卡"卡片
3. 验证：显示三项指标（🍎🍬🍞）
4. 点击"🍎 没有吃水果"
5. → 勾选框变绿，显示"✓ 已确认"
6. 刷新页面
7. 验证：打卡状态保持
8. 返回日历，点击该日期
9. → Modal 显示控糖打卡状态
10. 验证：显示"✓ 已确认"
```

### 场景4：数据隔离测试
```
1. 注册用户A：testuser0204
2. 创建计划，录入今日数据
3. 点击退出按钮
4. → 跳转到登录页
5. 注册用户B：testuser0205
6. 创建计划
7. 验证：Dashboard 无用户A的数据
8. 登出，登录用户A
9. 验证：看到自己的数据
10. 验证：用户A和B的数据完全独立
```

### 场景5：退出按钮测试

1. 登录后访问 /dashboard
